# XHS 对标账号分析（Competitor Analysis）

目标：搜索指定关键词或账号名，进入账号主页采集真实数据，输出一份可直接用于内容策略决策的竞品分析报告。

---

## ⚠️ 关键技术要点：小红书主页访问方式

**直接用数字小红书号拼 URL 会报「未连接到服务器」，必须先从搜索结果页抓取带 `xsec_token` 的完整 URL，再导航进入主页。**

---

## 1. 输入类型

用户可能提供：
- **账号名列表**：如「木子不写代码、RoRo用AI重做产品」
- **搜索关键词**：如「AI创业记录」「不写代码做产品」
- **赛道描述**：如「小红书上做AI工具图文的账号」

如果用户只给了赛道描述，先搜索 2-3 个关键词，找到 3-5 个相关账号后继续。

---

## 2. 采集流程（每个账号逐个执行）

### 第一步：搜索账号，抓取带 token 的 URL

用浏览器打开搜索页（按账号名搜索）：
```
https://www.xiaohongshu.com/search_result?keyword=账号名&type=user
```

等待 2-3 秒页面加载后，运行以下 JS 抓取所有账号链接：
```js
JSON.stringify(Array.from(document.querySelectorAll('a[href*="user/profile"]'))
  .map(a => ({href: a.href, text: a.innerText.substring(0, 60)}))
  .filter(x => x.text && !x.text.includes('我'))
  .slice(0, 10))
```

如果搜索无结果，尝试：
- 换关键词（如把「×」换成空格，或用账号简介里的关键词搜笔记）
- 搜索笔记关键词（去掉 `&type=user`）再从笔记结果页找账号链接

### 第二步：进入主页，抓取基础数据

使用上一步抓到的完整 URL 导航进入主页，等待 2-3 秒后运行：
```js
document.body.innerText.substring(0, 4000)
```

从文本中提取：
- 账号名、小红书号、IP 属地
- 粉丝数、关注数、获赞与收藏总数
- 简介内容
- 笔记列表：标题 + 紧跟在标题后面的数字（即赞藏数）

### 第三步：滚动加载更多笔记（可选）

如果只看到前 10 条笔记，可以 scroll down 加载更多，然后再次抓取页面文本。
建议采样最近 **9-15 篇**，兼顾效率与代表性。

### 第四步：进入 2-3 篇代表性笔记（可选，深度分析时使用）

- 优先选赞藏数最高的 1-2 篇 + 最近发布的 1 篇
- 记录：正文结构、封面文字、互动 CTA、评论区反应
- 仅从搜索结果页/主页卡片点击进入，禁止直接 navigate 到 `/explore/<id>`

---

## 3. 分析维度

对每个账号，从以下 5 个维度判断（结合 xhs-account-analysis.md 的五维框架）：

| 维度 | 关注点 |
|------|-------|
| 定位清晰度 | 账号讲什么，能否一句话说清 |
| 内容结构力 | 标题钩子、封面层级、正文节奏 |
| 互动转化力 | 评论区是否被引导、收藏/评论结构是否健康 |
| 账号辨识度 | 人设、语气、视觉是否统一 |
| 增长可持续性 | 爆款是否依赖偶发，还是有稳定结构 |

---

## 4. 内容类型分类（帮助归纳爆款规律）

| 类型 | 特征 |
|------|------|
| **痛点解决型** | ✕问题 → ✓解决方案，对比结构，收藏率高 |
| **工具实战型** | 「X分钟实现/搭建XXX」「我用XXX做了…」 |
| **干货模板型** | 「XXX怎么做？模板来了」「X个技巧/工具」 |
| **情绪共鸣型** | 「凌晨X点崩溃实录」「劝退｜这类人别做…」 |
| **里程碑型** | 「第X天」「发车了」「终于上线了」 |
| **身份反差型** | 普通背景 × 高端结果，反差制造悬念 |
| **观点型** | 「为什么…」「XX已死/结束了」 |

## 6. 报告保存

完成分析后：
1. 将报告写入：`~/clacky_workspace/xhs-competitor-analysis-图文型-YYYY-MM-DD.md`
2. 同时更新 `knowledge-base/accounts/` 目录（每个账号一条记录）
3. 提炼出的爆款公式沉淀到 `knowledge-base/patterns/`
4. 口头总结 2-3 个最重要发现（不超过 200 字）

---

## 7. 分析质量要求

- **爆款笔记**：页面中每条笔记标题后紧跟的数字即为赞藏数，重点关注数字特别大（几百到几千）的
- **关键发现要有观点**：不只是罗列数据，要有判断——这个账号最大的优势/弱点是什么
- **差异化建议要具体**：不要说「你可以做得更好」，要说「他没有X，而你有X，可以这样利用」
- **图文专项关注**：特别标注哪些账号以图文为主、哪些爆款是图文格式，为图文策略提供参考

---

## 8. 失败降级

- 某个账号搜索失败（找不到 token URL）→ 跳过，继续下一个，报告中注明「未能获取」
- 主页信息加载不全 → 先完成轻量诊断，不为补齐字段反复刷新
- 进详情页连续失败两次 → 退回主页卡片层分析，提醒用户当前抓取受限
- 样本偏少 → 明确标注「样本偏少，结论仅供参考」
