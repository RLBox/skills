# 飞书妙记增量同步

增量同步飞书妙记到日记：轮询飞书 Drive → 筛选最近 N 天妙记 → 跳过已有 → 下载新增 → AI 提炼 → 补写日记。

## 核心逻辑

妙记录音存在录音豆设备上，用户打开飞书客户端才会上传到云端。今天的妙记可能今天同步，也可能明后天才出现。因此每次运行覆盖最近 3 天，做**增量补漏**。

## 执行步骤

### 第一步：计算日期范围

```bash
for i in 0 1 2; do
  date -v-${i}d "+%Y-%m-%d"
done
```

同时准备飞书中文格式：`2026-04-27` → `2026年4月27日`（月日去掉前导零）。

### 第二步：拉取飞书 Drive 文件列表

```bash
lark-cli api GET /open-apis/drive/v1/files
```

从返回 JSON 中筛选：
- `name` 包含目标日期任意一天的飞书中文格式日期
- `name` 以「智能纪要：」或「文字记录：」开头

提取每个文件的 `name`、`token`，从 `name` 中解析对应的 YYYY-MM-DD 日期。

如果完全没有匹配文件，静默结束。

### 第三步：增量下载

目标路径格式：`feishu-notes/{date}/{name}.md`

对每个筛选到的文件，检查本地是否已存在：
```bash
test -f "feishu-notes/${date}/${name}.md"
```

- **已存在 → 跳过**，不重新下载，不重复处理
- **不存在 → 下载**：

```bash
lark-cli docs +fetch --doc <token>
```

将返回 JSON 中 `data.markdown` 字段保存为 Markdown 文件：
```markdown
# {name}

{markdown内容}
```

如果没有任何新下载文件，静默结束。

### 第四步：AI 提炼（只处理智能纪要）

对每个有**新下载智能纪要**的日期：
1. 读取该日期所有「智能纪要：」开头的 markdown 文件
2. 「文字记录：」文件不参与提炼
3. 提炼三个维度：

- **金句**（2-3 条）— 有价值的观点、有启发的表达，附说话人及场景
- **关键决策**（2-3 条）— 明确的决定、行动计划
- **核心要点**（3-5 条）— 重要的讨论主题、进展更新

要求：简洁直接，每条 1-2 句话。只提炼原文中明确的内容，不编造不推测。

### 第五步：补写日记

目标文件：`memory/{date}.md`（date 是妙记对应的日期，可能是今天/昨天/前天）。

1. 读取现有内容（如不存在则创建基础结构）
2. 定位 `## 从妙记提炼的关键信息` 区域：
   - **区域不存在** → 在文件末尾追加
   - **区域存在但为空或仅占位** → 替换内容
   - **区域已有实质内容** → 在区域内**最前面**追加新条目（新在前，旧在后）
3. 写入格式：

```markdown
## 从妙记提炼的关键信息

_YYYY-MM-DD HH:MM 自动同步，共处理 N 份智能纪要_

> 会议：XXX（HH:MM-HH:MM）、YYY（HH:MM-HH:MM）

### 金句
- 「引用原文」—— 简要标注说话人/场景

### 关键决策
- **决策主题**：具体内容

### 核心要点
- 要点描述
```

### 第六步：更新 MEMORY.md（可选）

从本次所有提炼结果中识别**重要经验、方法论、教训**，标准：
- ✅ 可复用的方法论
- ✅ 踩过的坑和解决方案
- ✅ 核心决策和原则
- ❌ 单纯的项目进展（只记日记，不进 MEMORY）

如有符合条件的内容，追加到 `MEMORY.md`。

### 第七步：静默完成

任务完成后无需输出多余内容。

## 重要提醒

- **增量原则**：本地已存在的文件必须跳过
- **补漏原则**：发现哪天的新妙记就补写哪天的日记（不限于昨天/今天）
- 飞书 Drive 返回的文件在同一层，按文件名里的日期筛选
- `lark-cli docs +fetch --doc <token>` 返回 JSON，取 `data.markdown` 字段
- 「智能纪要：」和「文字记录：」都要下载保存；只有「智能纪要：」参与提炼
- 文件名包含中文冒号「：」和空格，路径拼接时注意
